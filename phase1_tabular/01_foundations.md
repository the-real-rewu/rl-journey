## 1. The Markov Decision Process (MDP)

An MDP is a 5-tuple `(S, A, P, R, ╬│)`:

| Symbol | Name | What it is |
|--------|------|------------|
| `S` | States | The set of situations the agent can be in |
| `A` | Actions | The set of choices the agent can make |
| `P(s'\|s,a)` | Transition function | Probability of landing in `s'` after action `a` in state `s` |
| `R(s,a,s')` | Reward function | Scalar reward for that transition |
| `╬│ Ōłł [0,1]` | Discount factor | How much future rewards matter versus immediate ones |

**The Markov property.** "The future depends only on the present, not the
past." Formally: `P(s_{t+1} | s_t, a_t) = P(s_{t+1} | s_t, a_t, s_{t-1}, ...)`.
The current state is sufficient to predict the next one.

**Why this matters.** If the Markov property holds, we can make decisions
from the current state alone. If it doesn't, we're in a partially-observable
setting (POMDP) and life gets harder. GridWorld is cleanly Markov.

### GridWorld as an MDP

- `S` = all (row, col) positions that aren't walls
- `A` = {up, right, down, left}
- `P` = deterministic (prob 1 on one outcome). Off-grid moves stay put.
- `R` = -1 per step, +10 for goal, -10 for hazard
- `╬│` = your choice (try 0.9 or 0.99)

## 2. Policy, Trajectory, Return

### Policy `ŽĆ(a | s)`

A policy is a mapping from states to actions ŌĆö how the agent behaves.
- **Deterministic:** `ŽĆ(s) = a`
- **Stochastic:** `ŽĆ(a | s) = P(choose a in s)`

Most of this phase uses deterministic policies, but the equations are
written for stochastic policies (deterministic is just a special case
where one action has probability 1).

### Trajectory

A sequence generated by running the policy in the environment:
`s_0, a_0, r_1, s_1, a_1, r_2, s_2, ...`

Note the indexing: the reward `r_{t+1}` arrives *after* taking action `a_t`
in state `s_t`. Some textbooks use `r_t` instead. The math is the same;
just stay consistent.

### Return `G_t`

The total discounted reward from time `t` onward:

```
G_t = r_{t+1} + ╬│ r_{t+2} + ╬│┬▓ r_{t+3} + ...
    = ╬Ż (k=0 to Ōł×) ╬│^k ┬Ę r_{t+k+1}
```

**Why discount?** Three reasons:
1. **Math:** `╬│ < 1` keeps `G_t` finite even for infinite episodes.
2. **Model of reality:** A reward now is usually worth more than the same
   reward later (think interest rates, uncertainty about the future).
3. **Prioritization:** Low `╬│` makes the agent myopic (greedy for immediate
   reward); high `╬│` makes it far-sighted. It's a knob you tune.

## 3. Value Functions

The value function tells you "how good is it to be here (and do what)?"

### State-value function `V^ŽĆ(s)`

The expected return starting from state `s` and following policy `ŽĆ` thereafter:

```
V^ŽĆ(s) = E_ŽĆ [ G_t | s_t = s ]
```

Read aloud: "the expected total discounted reward if I start in state `s`
and act according to `ŽĆ` forever after."

### Action-value function `Q^ŽĆ(s, a)`

The expected return starting from `s`, taking action `a`, *then* following `ŽĆ`:

```
Q^ŽĆ(s, a) = E_ŽĆ [ G_t | s_t = s, a_t = a ]
```

Read aloud: "the expected total discounted reward if I start in `s`,
commit to action `a` right now, and then act according to `ŽĆ`."

### Why have both?

- `V` is compact ŌĆö one number per state.
- `Q` is actionable ŌĆö to pick the best action you don't need to know `P`;
  you just pick `argmax_a Q(s, a)`.
- If you know `V` and `P`, you can compute `Q`. If you know `Q`, you can
  recover `V = ╬Ż_a ŽĆ(a|s) Q(s, a)`.

**Practical consequence:** Model-free control methods (SARSA, Q-Learning)
learn `Q` directly, because they don't have access to `P`.

## 4. The Bellman Equations

This is the single most important idea in RL. Everything is a consequence
of this.

### Bellman expectation equation (for a fixed policy `ŽĆ`)

```
V^ŽĆ(s) = ╬Ż_a ŽĆ(a|s) ╬Ż_{s'} P(s'|s,a) [ R(s,a,s') + ╬│ V^ŽĆ(s') ]
```

In words: **the value of a state equals the expected immediate reward
plus the discounted value of the next state.**

The recursion is what makes this powerful. We've expressed `V^ŽĆ` in terms
of itself ŌĆö so learning `V^ŽĆ` becomes a fixed-point problem.

### Bellman optimality equation

Define the optimal value function `V*(s) = max_ŽĆ V^ŽĆ(s)` ŌĆö the value under
the best possible policy. Then:

```
V*(s) = max_a ╬Ż_{s'} P(s'|s,a) [ R(s,a,s') + ╬│ V*(s') ]
Q*(s, a) = ╬Ż_{s'} P(s'|s,a) [ R(s,a,s') + ╬│ max_{a'} Q*(s', a') ]
```

The key difference: a `max` replaces the expectation over `ŽĆ`. Once you
have `Q*`, the optimal policy is `ŽĆ*(s) = argmax_a Q*(s, a)`.

### Why these equations matter

- **Prediction** (given `ŽĆ`, find `V^ŽĆ`) reduces to solving the Bellman
  *expectation* equation. ŌåÆ Policy Evaluation (section 1.2).
- **Control** (find `ŽĆ*`) reduces to solving the Bellman *optimality*
  equation. ŌåÆ Value Iteration, Q-Learning (sections 1.2 and 1.4).
- Every algorithm in the next three sections is a way to solve these
  equations under different assumptions (model known vs unknown,
  on-policy vs off-policy, sampled vs exact).

Set up a 2├Ś2 GridWorld: start at (0,0), goal at (1,1), step reward -1,
goal reward +10, ╬│ = 0.9. Consider the trajectory:

```
s_0 = (0,0), a_0 = RIGHT, r_1 = -1, s_1 = (0,1)
s_1 = (0,1), a_1 = DOWN,  r_2 = 10, s_2 = (1,1) [terminal]
```

Compute `G_0` and `G_1`.

### Exercise 2 ŌĆö Value under a simple policy

Same environment. Consider the deterministic policy "always go RIGHT".
Starting from (0,0), the agent bumps the wall forever.

What is `V^ŽĆ((0,0))`? (Hint: it's an infinite geometric series of step rewards.)

Same environment. Policy: "RIGHT at (0,0), DOWN at (0,1)".
What is `V^ŽĆ((0,0))`? What is `V^ŽĆ((0,1))`? Verify they satisfy the
Bellman expectation equation.
